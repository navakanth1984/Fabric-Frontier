# ACDLC Release Notes - v1.5

Upgraded the platform from a governed lifecycle framework into a full **governed Agent Execution Kernel and operating platform** by decoupling path resolution into a service catalog model, establishing stateful runtime queues, creating isolated memory layers, building a policy compilation pipeline, implementing visual telemetry dashboards, and introducing automated safety simulation tests.

---

## 🚀 Enhancements & Core Features

- **Decoupled Service Catalog Registry**: Refactored `registry/manifest.yaml` to decouple logical skill paths from physical workspace resolutions (`root_resolution`). Expanded `schemas/registry-schema.json` to enforce status parameters (`active`, `experimental`, `deprecated`, `archived`) and dynamically auto-inject dates.

- **Distributed Execution Runtime (`runtime/`)**: Created a robust operational layer managing priority-weighted queues (`queue.py`), worker processing bounds (`worker.py`), Star-topology dependency scheduling (`scheduler.py`), state serialization checkpointers (`checkpoint.py`), and persistence syncing (`state_manager.py`).

- **Isolated Cognitive Memory Layer (`memory/`)**: Structured cognitive context budgets by separating **Episodic** history log traces, **Semantic** reusable schemas, and active **Working** context scopes, backed by formal schemas (`episodic-recall.json`, `working-context.json`).

- **Declarative Policy Compiler (`compile_policies.py`)**: Designed an automated compilation script translating human-readable YAML policies into optimized JSON rule graphs (`compiled-policies/`) to eliminate latency and guarantee execution determinism.

- **Visual Observability Dashboard (`generate_dashboard.py`)**: Synthesized telemetry CSV logs into a gorgeous responsive HTML/CSS dashboard (`analytics/dashboard.html`) mapping success rates, token burn rates, stage durations, and recovery loops.

- **Automated Sandbox Simulations (`simulations/`)**: Created a sandboxed stress testing module (`run_simulations.py`) verifying platform gates against 8 edge-case failure loops (`SIM-001` through `SIM-008`).
