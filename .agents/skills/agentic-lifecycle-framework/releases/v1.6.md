# ACDLC Release Notes - v1.6

Evolved the platform from an execution kernel into a **fully stateful Distributed Agent Runtime**, establishing event-driven execution loops, workload-specific priority queue lanes with SLAs, capability-aware routing, schema-enforced state versioning metadata, and partitioned telemetry log warehouses.

---

## 🚀 Enhancements & Core Features

- **Runtime Event Bus (`event_bus.py`)**: Integrated a thread-safe pub-sub message broker. All events published contain UUIDs (`event_id`), parent tracking correlation IDs (`correlation_id`), and ISO-8601 timestamps.

- **Multi-Queue Scheduling & SLAs**: Transitioned queue execution into dedicated workload priority SLA lanes:
  - `recovery` (P0 / SLA Max Wait 0s)
  - `validation` (P1 / SLA Max Wait 5s)
  - `execution` (P2 / SLA Max Wait 30s)
  - `research` (P3 / SLA Max Wait 300s)
  - Recovery tasks promoted dynamically to speed up self-healing preemption.

- **Capability-Aware Registry**: Extended skill service catalogs in `registry/manifest.yaml` to parse competencies (planning, research, writing, testing, optimizing, coding, validation), dynamically routed via the scheduler.

- **Enforced State Versioning**: Standardized lifecycle configurations with mandatory `schema_name`, `schema_version` (v1.0), and `generated_by` metadata properties to prevent migration conflicts.

- **Partitioned Telemetry Warehouse**: Structured `analytics/warehouse/` into partitioned database subfolders (`executions`, `policies`, `escalations`, `simulations`, `telemetry`), dynamically managed by the state manager wildcards.

- **4-Tab Observability Console (`dashboard.html`)**: Upgraded dashboard scripts compiling CSV records, registries, and warehouse events into a multi-tab JS visual console with active Policy Violation tables.

- **Preemption & Bus Simulations**: Expanded sandbox test runs (`run_simulations.py`) with event bus thread safety check (`SIM-009`) and multi-queue priority preemption SLA gates (`SIM-010`).
