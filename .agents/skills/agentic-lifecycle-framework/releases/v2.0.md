# Release v2.0

## Overview
Achieves the pinnacle architectural milestone: **Secure Multi-Tenant Agent OS**. With routing, scalability, persistence, and learning firmly established in previous versions, v2.0 introduces the comprehensive Security Subsystem required to safely host multiple isolated tenants and strictly gate model usage.

## Features
- **Authentication Engine**: `auth.py` validates Bearer tokens and translates credentials into identity records.
- **Authorization & Roles**: `roles.py` defines explicit permissions mapped to RBAC tiers (Viewer, Agent, TenantAdmin, SystemAdmin). `authorization.py` strictly prevents cross-tenant access and enforces permission checks.
- **Secrets Manager**: The `secrets_manager.py` enclave guarantees API keys are safely held and proactively obfuscated from entering the `EventStore` or Audit logs.
- **Audit Trail**: Every access evaluation (ALLOW/DENY/ESCALATE) is durably recorded into an immutable JSONL log, serving as a forensic ledger for the cluster.
- **Runtime Security Policies**: `policies.py` evaluates tenant-level model entitlements, completely cutting off execution paths if a basic tier attempts to utilize premium endpoints like `GPT-Ultra`.
