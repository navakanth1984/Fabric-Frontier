# Workflows: Stage 6 - Observation & Evolution

Agentic systems are not static. The Observation stage serves as the continuous learning loop, gathering telemetry and refining ACDLC processes for the next cycle.

### Step 1: Collect Telemetry Metrics
Assemble stats from the completed run:
- Total execution time and latencies.
- Retry counts per task.
- Hallucination and syntax failure rates.
- Total token expenditure and financial costs.

### Step 2: Extract Lessons & Bottlenecks
Analyze telemetry to discover inefficiencies:
- *Did a subagent struggle to read a particular documentation layout?* (Context compression error)
- *Did the reasoning economics budget fail due to using Sonnet/Pro on a simple syntax task?* (Routing scheduler error)
- *Were there recurring test failures due to ambiguous specification rules?* (Stage 0 ALIGN error)

### Step 3: Refine the System
Based on findings, execute dynamic updates:
1. **Update Context Bundles**: Add missing rules or clarify schemas.
2. **Optimize Prompts**: Refine agent instructions to prevent observed failure modes.
3. **Refine Skills**: Modify `SKILL.md` or workflow definitions to integrate new optimizations.

### Step 4: Generate vNext Blueprint
Compile recommendations for the next feature loop, defining a refined ACDLC vNext configuration.
